# 訓練推理模型：RLVR 與 GRPO

前沿的推理模型（OpenAI 的 o 系列、DeepSeek-R1、Qwen 的 thinking 模型）採用的訓練配方，跟 [RLHF 與 DPO](04-rlhf-and-dpo.md) 介紹的 RLHF 和 DPO 不同：**可驗證獎勵的強化學習（reinforcement learning with verifiable rewards，RLVR）**。做法是拿一個有能力的基礎模型，讓它在具有*可檢查*答案的問題上（數學、程式碼、邏輯）產生長串的思維鏈，然後用程式化的驗證器直接獎勵答案的正確性，而不是用一個學習得來的獎勵模型。本章涵蓋 RLVR、DeepSeek 帶起風潮的 GRPO 演算法、獎勵設計的失敗模式、關於 RL 究竟真正帶來了什麼的當前爭論，以及多數團隊應該優先採取的較便宜的蒸餾路徑。

## RLVR：可驗證獎勵的強化學習

RLVR 把 RLHF 那個學習得來的獎勵模型，換成一個確定性的**驗證器**，由它根據真實答案的檢查回傳獎勵：最終答案是否相符（數學）、程式碼是否通過單元測試、證明是否能通過檢查。這裡的策略就是 LLM；環境是一次單回合的生成，先產生一長串思維鏈，再給出答案；獎勵則由數值/字串比對，或實際執行程式碼來計算。

**為什麼「可驗證」很重要：它在很大程度上繞過了獎勵模型被鑽漏洞（reward-model hacking）的問題。** 在 RLHF 裡，獎勵模型本身就是一個用偏好資料訓練出來的神經網路，而策略會學著去鑽它的盲點（這就是 Goodhart／過度最佳化問題）。而一個「這是否等於標準答案」或「是否所有測試都通過」的驗證器，沒有那種柔軟、可學習、能用同樣方式被鑽的表面；它的獎勵是接地（grounding）在一個外部的真理判準上。這就是 RLVR 在推理任務上比 RLHF 擴展得更穩定的核心原因。它並沒有*消除*鑽漏洞的可能（見[獎勵設計](#reward-design-and-failure-modes)），但它移除了最容易的那種鑽法。

每一步的迴圈：
1. 取樣那些已知可檢查答案的提示。
2. 對每一個提示，從當前策略取樣一個或多個長思維鏈的補全。
3. 對每個補全執行驗證器，得到一個純量獎勵（通常是正確 1／錯誤 0，有時再加上一個小小的格式獎勵）。
4. 計算優勢值（advantage），並進行一次策略梯度更新。
5. 重複。長串的思維鏈、自我驗證與回溯，都是從這個迴圈中*湧現*出來的，而不是被明確監督出來的。

最具代表性的公開成果是 **DeepSeek-R1**（arXiv:2501.12948；經同儕審查的版本於 2025 年 9 月發表於 Nature），它報告指出，光靠*純 RL*、完全不需要人工標註的推理軌跡，就能誘發出推理能力，並伴隨湧現出的自我反思。一個教學上的提醒：OpenAI 並未公開 o 系列的演算法，所以「o 系列採用 RLVR 風格的 RL」只能當成從公開訊息推得的合理推論，而非有文件佐證的事實，也不要把任何特定演算法歸給它。

## GRPO

**群組相對策略最佳化（Group Relative Policy Optimization）** 是在 DeepSeekMath（arXiv:2402.03300）中提出的，並被用作 DeepSeek-R1 的 RL 引擎。

**它移除掉的那個 PPO 問題。** PPO 需要一個**價值/評論家網路（value/critic network）**，通常跟策略一樣大，用來估計計算優勢值時所需的基準線（baseline）。對 LLM 來說，這大致會讓記憶體加倍，並多出第二個模型要訓練，而且要為一個只有單一終端獎勵的長序列學出價值估計也很困難。GRPO 的做法是：**刪掉評論家。** 它不用學習得來的基準線，而是針對同一個提示取樣一**群（group）**補全，並用這群補全自己的獎勵統計量當作基準線。

**群組相對優勢值。** 對一個提示，取樣補全 `o_1..o_G`，得到獎勵 `r_1..r_G`。補全 `i` 的優勢值就是它在這個群組內的 z 分數，並廣播（broadcast）到該補全的每一個 token 上：

```python
# One GRPO step. policy = the LLM; verify() = a programmatic checker.
for q in sample_prompts(batch):
    group = [policy.generate(q) for _ in range(G)]      # G group, e.g. 8..64
    R = [verify(q, o) for o in group]                    # verifiable reward, e.g. 1.0 / 0.0
    mu, sigma = mean(R), std(R)
    A = [(r - mu) / (sigma + 1e-6) for r in R]           # group-relative advantage (the critic-free baseline)
    for o_i, A_i in zip(group, A):
        ratio = exp(policy.logprob(o_i) - old_policy.logprob(o_i))  # per-token prob ratio
        loss += -mean(min(ratio * A_i, clip(ratio, 1-eps, 1+eps) * A_i))
        loss += beta * KL(policy, ref_policy)            # original GRPO keeps a KL term
update(policy, loss)
```

DeepSeek 之所以採用它，是因為在前沿規模下，捨棄評論家是實際的記憶體勝利，而且這種群組基準線很適合單一終端的可驗證獎勵。

**已知的問題，以及修正它們的變體：**
- **長度偏誤（Length bias）。** 由於一個純量優勢值被廣播到每一個 token 上，較長的補全會把它的優勢值攤分到更多 token，於是最佳化過程可能會把回應長度灌水（對錯誤答案尤其如此）。**Dr.GRPO**（arXiv:2503.20783）移除了造成這個現象的長度與標準差正規化項。
- **零變異崩潰（Zero-variance collapse）。** 如果一個群組裡所有補全都拿到相同的獎勵（在簡單題上全對，或在困難題上全錯），標準差就會趨近於零，每個優勢值都變成零，這個群組也就*貢獻不出任何梯度*。隨著模型變強，越來越多提示變成「全對」，於是有效批次（effective batch）會縮水。**DAPO**（arXiv:2503.14476）以**動態取樣（dynamic sampling）**修正了這點：先過量取樣，再把任何群組全對或全錯的提示濾掉，只保留那些能產生梯度的群組。DAPO 也把裁切範圍解耦（「clip-higher」）、改用 token 層級的損失，並捨棄了 KL 項。
- **MoE 不穩定。** 對混合專家（mixture-of-experts）模型而言，token 層級的重要性比值很吵雜，因為不同策略版本之間的路由（routing）會不一樣。**GSPO**（arXiv:2507.18071，用於 Qwen3）改為以整段序列的概似度來定義比值，從而為 MoE 穩定了 RL。

實務上的教訓：把提示策展在模型的難度帶上，而且相較於原版 GRPO，優先選用去偏的變體（Dr.GRPO、DAPO 或 GSPO）。

## 獎勵設計與失敗模式

即使有了可驗證的獎勵，策略最佳化的仍然是驗證器*實際*量到的東西，而不是你心裡想的那個目標。

- **鑽驗證器的漏洞。** 模型可以在沒學會該模式的情況下通過檢查器（例如，逐一列舉實例層級的標籤），這是一種有文件記載的 RLVR 獎勵鑽漏洞模式。
- **「奇蹟步驟」（Miracle steps）。** 一條思維鏈可以透過*不忠實*的中間步驟到達正確的最終答案（arXiv:2510.07774）。一個純粹只看結果的驗證器，會連同那個正確答案以及沿途糟糕的推理一起獎勵下去。
- **格式獎勵 vs 正確性獎勵。** R1-Zero 風格的配方，會把一個小小的格式獎勵（把推理放進帶標籤的區塊、給出可解析的答案）跟正確性獎勵搭配在一起。如果格式獎勵相對於正確性太大，模型就會去最佳化那個便宜、好鑽的訊號。
- **長度的利用**，既包含上面提到的 GRPO 梯度產物，也包含真正的「冗長度與通過率相關」這種鑽法。緩解手段：token 層級的損失、對過長回應的懲罰、長度正規化的目標函式。

一個需要特別標注、很重要的提醒：**Spurious Rewards** 的發現（arXiv:2506.10947）指出，在某一個模型家族上，用*隨機*甚至*錯誤*獎勵的 GRPO 竟然帶來了大幅的提升，原因是放大了該家族在預訓練中早已潛藏的某種行為。關鍵在於，這篇論文強調這些提升**並沒有**遷移到其他模型家族。教訓就是：對單一模型家族的 RLVR 結果要抱持高度懷疑，並且永遠要納入一個來自不同家族的基線。

## RL 是增加能力，還是讓取樣更銳利？

這是本章最細緻的一點：一個真正開放的問題，有三種立場。

- **RL 是讓取樣更銳利，而非增加能力。** 有一條研究路線（arXiv:2504.13837）報告指出，RLVR 模型在 pass@1 上勝過基礎模型，但在大的 pass@k 上基礎模型卻能追平甚至超越它們，這暗示 RL 模型用到的每一條推理路徑其實在基礎模型裡早就存在；RL 只是提高了取樣到正確思維鏈的機率，卻同時縮窄了可探索的範圍。
- **延長的 RL 會擴張邊界。** 反方的結果（ProRL，arXiv:2505.24864；以及 arXiv:2506.14245，它引入了一個能感知推理的 CoT-Pass@K 指標）報告指出，只要 RL 做得夠久，模型就能解出基礎模型在每一個 k 上都失敗的問題，而且 RLVR 改善的是*推理正確*的答案，不只是運氣好的最終結果。
- **兩者都對，分成兩個階段（這是要拿來主打的綜合論）。** 一個調和性的觀點（arXiv:2510.04028）發現，早期的 RL 是在利用與銳化（pass@1 上升，pass@k 持平），而延長的 RL 則是在探索並擴張邊界。最乾淨的實務綜合論（arXiv:2512.07783）報告指出，RL 帶來真正的提升（以高 pass@k 衡量）**只有在預訓練留下了餘裕、而且 RL 資料瞄準模型能力邊緣的時候**才會發生，也就是那些它在 pass@1 上會失敗、但在 pass@k 上偶爾能命中的問題。對已經解得出來或根本無望的問題做 RL，收穫甚微。

同一份研究也凸顯了**中段訓練（mid-training）**：這是介於通用預訓練與 RL 之間的一個階段，會在一個策展過、推理密度高的混合資料上繼續訓練，以鞏固正在湧現的技能。在固定的計算預算下，它往往勝過只做 RL，因為它抬高了那個之後要被 RL 銳化的基礎能力。誠實的解讀是：這個領域已經收斂到「RL 在能力邊緣運作得最好」的共識，但 RL 究竟能不能製造出*真正全新*的能力，仍有爭議。

## 蒸餾：較便宜的路徑

對多數團隊來說，RL 是錯誤的第一步。

**離策略蒸餾（off-policy distillation，在老師的軌跡上做 SFT）。** DeepSeek-R1 報告指出，他們在大約 80 萬條由 R1 生成的推理軌跡上微調了密集型的 Qwen 與 Llama 模型，而且蒸餾出來的 32B 模型，勝過直接在同一個 32B 基礎上做大規模 RL 的結果，論文並指出後者要花上多得多的計算量，卻仍然追不上蒸餾。教訓是：對於一個在已知領域上的小模型，從一個強推理模型蒸餾過來，比自己做 RL 更有效、也便宜得多。

**在策略蒸餾（on-policy distillation）**（Thinking Machines Lab，2025）是在學生*自己*取樣出來的補全上訓練學生（在策略，像 RL 一樣），但用的是來自老師分布的、密集的逐 token 監督（像 SFT 一樣），而不是稀疏的純量獎勵。其報告的結果是，在灌輸推理能力上，大約比 RL 的取樣效率高出一個數量級，結合了在策略的相關性與密集、穩定的訊號。截至 2026 年，這些數字大多來自單一廠商，所以那個確切的倍數要當成「報告值」，而非「已確立的事實」。

| 方法 | 軌跡 | 獎勵訊號 | 成本／穩定度 |
|--------|-------------|---------------|------------------|
| SFT／離策略蒸餾 | 老師的 | 密集的 token 目標 | 便宜、穩定；有一些曝光偏誤 |
| RL／RLVR | 學生自己的 | 稀疏的純量 | 昂貴、較不穩定；在策略 |
| 在策略蒸餾 | 學生自己的 | 密集（老師逐 token） | 兼具在策略的相關性與密集訊號 |

更廣泛的論述見[知識蒸餾](05-knowledge-distillation.md)。

## 實務指引

**自己訓練推理模型 vs 直接使用現成的。** 對大約 95% 的團隊來說：直接用一個強推理模型，或從中蒸餾。只有在以下情況才自己做 RLVR：(a) 你的領域有一個乾淨的程式化驗證器，(b) 基礎模型或老師模型確實無法被誘導出你的目標行為，(c) 你具備 RL 基礎設施、GPU 預算以及對不穩定性的容忍度，並且 (d) 你的任務正好坐落在模型能力的邊緣。

**粗略的需求量（這是數量級，不是承諾）：** 蒸餾想要 10^5 到 10^6 條高品質的老師軌跡，以標準 SFT 的計算量進行；RLVR 想要數千到數萬個策展到難度帶上的*可驗證*提示，而成本瓶頸在於 rollout（而非梯度步驟）。DeepSeek-R1 的後訓練據報約耗費 14.7 萬個 H800 GPU 小時，遠遠超出業餘玩家的規模，但比一般想像的少。

**評估的陷阱：** 同時回報 pass@1 *和* pass@k（光看 pass@1 會掩蓋掉取樣與能力之間的區別）；使用多個基礎家族（Spurious Rewards 那個產物）；檢查推理的忠實度，而不只是看最終答案；把回應長度的漂移當成獎勵鑽漏洞的警報來盯；並且優先採用保留集或全新的競賽題組，以防範基準測試的污染。見[基準測試與排行榜](../14-evaluation-and-observability/03-benchmarks-and-leaderboards.md)。

**成熟度：** 針對數學與程式碼、具備乾淨驗證器的蒸餾與 GRPO/DAPO 配方，已經成熟且可重現。在策略蒸餾、給 MoE 用的 GSPO，以及把中段訓練當成一個刻意安排的階段，都在快速成熟中。至於 RLVR 是否增加新能力，以及如何在不重新引入獎勵模型鑽漏洞的前提下，乾淨地把它延伸到可驗證領域之外，這些都還真正懸而未決，所以請把錨點放在那些機制上（群組基準線、長度正規化、動態取樣、密集 vs 稀疏監督、能力邊緣的資料），而不是任何單一的排行榜數字。

## 面試問題

### Q：什麼是 RLVR，以及為什麼它在推理任務上比 RLHF 擴展得更穩定？

**好的回答：**
RLVR 是可驗證獎勵的強化學習：你不是用一個學習得來的獎勵模型，而是用一個確定性的驗證器來獎勵策略，看數學答案是否相符、程式碼是否通過它的測試。它之所以比 RLHF 擴展得更穩定，是因為獎勵接地在一個外部的真理判準上，而不是接地在一個策略可以過度最佳化的神經網路上。RLHF 的經典失敗就是獎勵模型被鑽漏洞，也就是策略找到了那些獎勵模型給高分、但人類不會給高分的輸入；而一個單元測試的通過，並沒有可供類比利用的柔軟表面。代價是 RLVR 只適用於你能機械化地檢查正確性的地方，這也是為什麼它最先在數學、程式碼與邏輯上起飛。它並沒有消除鑽漏洞，模型仍然可以透過不忠實的步驟到達正確答案、或去鑽檢查器的漏洞，所以你還是得盯著忠實度與長度。

### Q：解釋 GRPO 以及它的其中一個已知失敗模式。

**好的回答：**
GRPO 捨棄了 PPO 的價值/評論家網路（對 LLM 來說那大致會讓記憶體加倍），並用一個群組相對的基準線取代學習得來的基準線：對同一個提示取樣好幾個補全，用驗證器給每一個評分，然後把群組內的 z 分數當成優勢值，廣播到每一個 token。這讓它既便宜、又很適合單一終端的可驗證獎勵，這也是 DeepSeek 採用它的原因。一個關鍵的失敗模式是零變異崩潰：當一個群組裡每個補全都拿到相同的獎勵（在簡單題上全對，或在困難題上全錯），標準差就是零，優勢值也是零，這個群組就產生不出任何梯度。隨著模型變強，越來越多提示變成全對，於是有效批次會縮水。DAPO 的動態取樣修正了這點，做法是濾掉全對與全錯的群組，只在那些真正能產生梯度的群組上訓練。此外，把一個優勢值廣播到所有 token 還會帶來長度偏誤，Dr.GRPO 透過移除長度正規化來處理它。
