# 微調策略

微調讓預訓練模型適應特定任務、領域或風格。如今，微調比較不是為了「教會事實」，而是為了「教會格式與行為」。

## 何時該微調 {#when-to-fine-tune}

在微調之前，先問自己：**這件事能不能用提示工程或 RAG 解決？**

| 需求 | 更好的解法 | 原因 |
|-------------|-----------------|-----|
| 新事實／知識 | **RAG** | LLM 不擅長從微調中記住事實；RAG 更容易更新。 |
| 特定輸出格式 | **微調** | 教會模型穩定輸出 JSON／XML，不需要複雜的提示。 |
| 語氣／人設 | **微調** | 比系統提示一致得多。 |
| 降低延遲 | **微調** | 減少對冗長 few-shot 提示的需求。 |
| 私有領域語言 | **持續預訓練** | 教會專業詞彙（醫療、法律、自訂程式碼）。 |

## 監督式微調（SFT） {#supervised-fine-tuning}

預訓練之後的第一步。模型在 `(Prompt, Response)` 配對上進行訓練。

### 品質階層
**1,000 筆「完美」範例勝過 1,000,000 筆雜訊範例。**
- **黃金資料集（Golden Sets）：** 由領域專家手工精選（技術任務需博士等級）。
- **負面約束訓練（Negative Constraint Training）：** 納入模型**不該**做什麼的範例（例如「不要道歉」、「不要提到自己是 AI」）。

---

## 持續預訓練（領域適應） {#continued-pretraining}

也稱為「第二階段預訓練」。
- **做法**：在特定領域的原始文字上訓練（例如，給金融模型訓練所有 SEC 申報文件）。
- **目標**：學習該領域語言的統計分布。
- **細節**：需要低得多的學習率（約為原始的 1/10），以避免「災難性遺忘」。

---

## PEFT 與全參數微調的比較 {#peft-vs-full-parameter}

| 特性 | 全參數微調 | PEFT（LoRA、QLoRA） |
|---------|-------------------|--------------------|
| GPU VRAM | 非常高（模型大小 * 4-12） | 低（模型大小 * 1.5） |
| 速度 | 基準 | 快 2x-3x |
| 風險 | 高（災難性遺忘） | 低 |
| 部署 | 每個任務一個模型 | 一個基礎模型 + 多個適配器 |
| **結論**| 保留給基礎模型訓練 | **生產環境標準** |

## 超參數調校 {#hyperparameter-tuning}

### 1. 學習率（LR）
- **SFT**：`1e-5` 到 `5e-5` 是標準值。
- **太高**：模型會「崩潰」，開始重複輸出或講出無意義的內容。

### 2. LoRA 的秩（r）
- 較高的秩（`r=64` 到 `r=256`）用於複雜的推理任務。
- 較低的秩（`r=8`）用於簡單的風格／語氣調整。

### 3. 打包訓練（Packing）
為了最大化吞吐量，我們把多筆較短的範例「打包」進單一條 4k 或 8k 序列，並以 EOS token 分隔。
- **挑戰**：自注意力可能會跨範例洩漏。
- **解法**：**搭配 block-masking 的 FlashAttention**，以防止跨範例的注意力。

## 面試題 {#interview-questions}

### Q：為什麼要用持續預訓練，而不是直接把領域資料放進 SFT 資料集？

**有力的答案：**
就資料建立而言，SFT 很「昂貴」，因為你需要提示／答案配對。持續預訓練讓你能利用大量原始、未標註的領域文字，教會模型的內部表徵專業詞彙與風格。一旦模型「會講這套語言」，你就用一個小型 SFT 資料集，教它用這套語言處理「任務」（例如分類、摘要）。

### Q：你如何避免模型在微調過程中「遺忘」通用能力？

**有力的答案：**
這就是「災難性遺忘」。兩種主要的緩解方式：
1. **複習（Rehearsal）：** 在你的微調資料集中混入 5-10% 的原始預訓練資料。
2. **PEFT（LoRA）：** 由於我們只訓練一小部分權重，原始的「知識」仍凍結在基礎模型權重中，大幅降低遺忘的風險。
