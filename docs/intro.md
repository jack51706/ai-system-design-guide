# AI System Design Guide

The living reference for production AI systems and staff-level interview prep. It covers RAG architectures, LLM engineering, agentic AI, MCP and A2A protocols, evaluation, and real interview questions, with concrete tradeoffs instead of vendor-neutral hedging.

This site is bilingual. Use the language dropdown in the top navigation bar to switch between English and 繁體中文.

## Pick a path

- **Prepare for interviews**: start with the [Question Bank](./00-interview-prep/01-question-bank.md), then [Answer Frameworks](./00-interview-prep/02-answer-frameworks.md).
- **Learn AI systems fast**: [LLM Internals](./01-foundations/01-llm-internals.md), then [RAG Fundamentals](./06-retrieval-systems/01-rag-fundamentals.md).
- **Build production RAG**: [Chunking](./06-retrieval-systems/02-chunking-strategies.md), [Vector Databases](./06-retrieval-systems/04-vector-databases.md), [Reranking](./06-retrieval-systems/06-reranking-strategies.md), [Production RAG at Scale](./06-retrieval-systems/14-production-rag-at-scale.md).
- **Build agents**: [Agent Fundamentals](./07-agentic-systems/01-agent-fundamentals.md), [Tool Use and MCP](./07-agentic-systems/03-tool-use-and-mcp.md), [Multi-Agent Orchestration](./07-agentic-systems/04-multi-agent-orchestration.md).
- **Pick a model (2026)**: [Model Taxonomy](./02-model-landscape/01-model-taxonomy.md), then [Pricing and Costs](./02-model-landscape/03-pricing-and-costs.md).
- **Evaluate AI in production**: [LLM Evaluation](./14-evaluation-and-observability/01-llm-evaluation.md) and [Benchmarks and Leaderboards](./14-evaluation-and-observability/03-benchmarks-and-leaderboards.md).

## How this guide is organized

The material follows the AI system lifecycle: **Foundations** (LLM internals, model landscape, training), **Build** (prompting, retrieval, agents, tool use, voice, multimodal), **Operate** (inference, memory, frameworks, infrastructure), **Govern** (security, reliability, evaluation), and **Apply** (design patterns, case studies, interview prep).
